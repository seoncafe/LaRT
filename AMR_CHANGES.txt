  octree_mod.f90 / amr_grid_type:
    - Added xrange, yrange, zrange fields (= xmax-xmin etc.) alongside the

      remove the implicit cubic-box assumption from AMR photon generation code.
    - Set in amr_build_tree alongside the existing xmin/xmax/ymin/ymax/zmin/zmax

    - grid_mod_amr.f90: amr_sync_to_grid now copies amr_grid%xrange/yrange/zrange

  generate_photon.f90 — AMR source geometry coverage:
    Previously, AMR mode only supported 'point', 'uniform', 'uniform_sphere'/
    'sphere', and 'diffuse_emissivity'. The following geometries are now also
    handled in AMR mode (using amr%xrange/yrange/zrange and amr%cx(1)/cy(1)/cz(1)
    for the box range and centre):
      - 'uniform_cylinder' / 'cylinder'
      - 'uniform_xy'
      - 'gaussian'
      - 'exponential'
      - 'exponential_sphere'
      - 'exponential_cylinder'
      - 'sersic' / 'ssh'
      - 'star_file'
    'plane_illumination', 'stellar_illumination', and 'point_illumination' are
    not yet supported in AMR mode; they fall back to a point source at
    (par%xs_point, par%ys_point, par%zs_point). These require dedicated AMR
    implementations of the underlying illumination routines.

Update note (2026-04-25):
Three coordinate-system and physics bugs were found and fixed. The generic FITS
reader incorrectly shifted leaf coordinates by the ORIGINX/Y/Z header values,
displacing the grid centre from (0,0,0) to (boxlen/2,...). The RAMSES reader
mapped fractional positions [0,1] to [0, boxlen_cm] instead of the LaRT
convention [-boxlen_cm/2, +boxlen_cm/2]. And amr_build_tree was called with
wrong box bounds [0, boxlen_code] instead of [-boxlen_code/2, +boxlen_code/2],
misplacing the octree root. Full details are in "Bugs Fixed (Coordinate System
/ Physics)" below.

Update note (2026-04-23):
This revision adds Stokes polarization support to AMR scattering and peel-off
routines, bringing AMR mode to full parity with the Cartesian Stokes path.

  scattering_amr.f90:
    scatter_dust_stokes_amr(photon, grid)
        Dust scattering with full Stokes vector update. Samples new polar angle
        from the scattering phase function via rand_alias_linear (alias method),
        selects azimuthal angle via a Stokes-weighted rejection method using
        S12/S11. Updates I, Q, U, V and rotates the reference frame triad
        (m, n, k) accordingly. Uses amr_grid%Jabs/Dfreq/vf* for cell data.

    scatter_resonance_stokes_amr(photon, grid)
        Resonance scattering with full Stokes vector update. Calls do_resonance
        procedure pointer with optional S11,S22,S12,S33,S44 output. Selects
        azimuthal angle by Stokes-weighted rejection. Supports core-skip
        (amr_grid%xcrit/xcrit2) and recoil correction (amr_grid%Dfreq(il)).
        Updates I, Q, U, V and triad vectors.

  peelingoff_amr.f90:
    peeling_dust_stokes_amr(photon, grid)
        Dust peel-off with Stokes I/Q/U/V output. Computes scattering angles
        (cost/sint) and azimuthal angle (cosp/sinp) toward the observer,
        looks up the scattering matrix elements (S11/S12/S33/S34) via
        interp_eq, applies the IAU frame rotation (cos2g/sin2g) to align
        Stokes parameters with the detector plane, and accumulates I/Q/U/V
        in both 2D and 3D observer arrays.

    peeling_resonance_stokes_amr(photon, grid)
        Resonance peel-off with Stokes I/Q/U/V output. Computes the resonance
        scattering matrix (S11/S22/S12/S33/S44) from photon%E1/E2/E3 (set by
        do_resonance). Applies Doppler frequency shift using amr_grid%Dfreq(il),
        rotates Stokes to the detector frame, and accumulates I/Q/U/V.

  setup.f90 (AMR block):
    scatter_dust/resonance now dispatches to stokes_amr or nostokes_amr based
    on par%use_stokes (previously always assigned nostokes_amr).
    peeling_dust_stokes and peeling_resonance_stokes now point to the new
    AMR-native routines instead of the Cartesian fallbacks.

Update note (2026-04-20):
This revision adds RAMSES RMHD layout support to the binary AMR reader.
If `output_NNNNN/hydro_file_descriptor.txt` is present, the reader now
detects whether hydro variables 2--4 are stored as momentum densities or
velocities, and whether the thermodynamic variable is total energy density or
thermal pressure. The AMR header skip count was also corrected to match the
IDL `rd_amr.pro` reader (`headf`, `ordering`, Hilbert keys, coarse `son`,
`flag1`, `cpu_map` -> six records after `ngridlevel` / boundary arrays).
In addition, a standalone Fortran converter,
`convert_ramses_to_generic_fits.f90`, was added to export RAMSES snapshots to
generic AMR FITS or FITS.GZ tables.

Update note (2026-04-19):
This revision adds diffuse-emissivity source support in AMR mode and refactors
the Cartesian diffuse-emissivity setup into a dedicated helper subroutine so
that Cartesian and AMR injection paths now have matching high-level structure.
It also extends the generic AMR reader so that if `par%amr_file` ends with
`.fits` or `.fits.gz`, LaRT reads the file as an AMRGrid FITS binary table
using the FITS I/O helper routines. For the FITS table, the generic reader
uses the column order (x, y, z, level, gas density, T, vx, vy, vz) rather
than matching specific column names.

--------------------------------------------------------------------------------
Overview
--------------------------------------------------------------------------------
This document describes the changes made to extend LaRT_v2.00 with octree-based
AMR grid support. The extension includes reading RAMSES simulation data and a
generic text-format reader that allows arbitrary octree structures to be
supported by providing a custom reader subroutine.

The design uses Fortran procedure pointers so that existing modules
(run_simulation_mod, gen_photon_car, etc.) require no modification.
When par%use_amr_grid = .true., the procedure pointers are reassigned to the
AMR variants at startup.

New Files

1. octree_mod.f90
   Module: octree_mod

   Defines the AMR grid type (amr_grid_type) and the global variable amr_grid.

   amr_grid_type fields:
     Tree structure (MPI-3 shared memory, one copy per node):
       ncell              -- total number of cells in the octree
       nleaf              -- number of leaf cells
       parent(:)          -- parent cell index (1-based; 0 = root)
       children(8,:)      -- eight child cell indices per cell (0 = absent)
       level(:)           -- refinement level of each cell
       cx(:), cy(:), cz(:)-- cell centre coordinates (code units)
       ch(:)              -- cell half-width (code units)
       ileaf(:)           -- leaf index per cell (>0 = leaf; 0 = internal)
       icell_of_leaf(:)   -- cell index for each leaf

     Physical arrays (size nleaf, MPI-3 shared memory):
       rhokap(:)          -- HI line-centre opacity per unit length
       rhokapD(:)         -- dust opacity per unit length (if DGR > 0)
       Dfreq(:)           -- local Doppler frequency width (Hz)
       voigt_a(:)         -- Voigt damping parameter
       vfx(:), vfy(:), vfz(:) -- bulk velocity / v_thermal (dimensionless)
       Pem(:)             -- diffuse-emissivity PDF (shared; only if emiss_file set)
       Pwgt(:)            -- composite-bias sampling weight per leaf
       alias(:)           -- alias table for leaf-based emissivity sampling

     Frequency grid (MPI-3 shared memory):
       xfreq_min, xfreq_max, dxfreq, nxfreq
       Dfreq_ref          -- reference Doppler frequency
       xfreq(:)           -- frequency array (Doppler units)
       velocity(:)        -- velocity array (km/s)
       wavelength(:)      -- wavelength array (Angstrom)

     Output accumulation arrays (size nxfreq, per-rank allocatable):
       Jout(:)            -- escaping photon spectrum
       Jin(:)             -- injected photon spectrum
       Jabs(:)            -- absorbed photon spectrum

     Core-skip parameters:
       xcrit, xcrit2      -- critical frequency and its square

     Box geometry:
       box_xmin/xmax, box_ymin/ymax, box_zmin/zmax  (cm)
       boxlen             -- box side length (cm)
       voigt_amean        -- mean Voigt damping parameter

   Key subroutines/functions:
     amr_find_leaf(x,y,z)
         Returns the leaf index for the given position.
         Descends the octree from the root by comparing (x,y,z) against
         each node's centre until a leaf is reached.

     amr_find_cell(x,y,z)
         Returns the cell index (internal use).

     amr_cell_exit(icell, x,y,z, kx,ky,kz, t_exit, iface)
         Computes the exit time t_exit and exit face iface for a ray
         starting at (x,y,z) with direction (kx,ky,kz) inside icell.
         Six-face slab intersection: t = (face - coord) / direction.
         Face index convention: 1=+x, 2=-x, 3=+y, 4=-y, 5=+z, 6=-z.

     amr_build_tree(xleaf,yleaf,zleaf, leaf_level, nleaf, box_*min/max)
         Builds the full octree from a list of leaf cell positions and
         levels. Inserts each leaf by descending from the root and
         creating intermediate nodes as needed. Uses grow_local_arrays
         to resize local temporaries dynamically, then copies the
         completed tree topology to MPI-3 shared memory.

     amr_build_neighbors()
         Builds a precomputed face-neighbor table (6 entries per cell,
         size 6×ncells) stored as MPI-3 shared memory. Used by
         amr_next_leaf for O(1) cell boundary crossings without tree
         re-descent.

     amr_next_leaf(il, iface)
         Returns the neighbor leaf index across face iface of leaf il,
         using the precomputed neighbor table.

     amr_alloc_phys(use_dust)
         Allocates rhokap, rhokapD (if use_dust), Dfreq, voigt_a,
         vfx, vfy, vfz as MPI-3 shared-memory arrays (size nleaf,
         one copy per shared-memory node). Only h_rank==0 fills them.

     amr_alloc_output(save_jin, save_jabs, use_dust)
         Allocates Jout (always), Jin (if save_jin), Jabs (if save_jabs)
         as per-rank regular allocatables (NOT shared memory), size nxfreq.

     grow_local_arrays(new_max, old_max, ...)
         Private helper used only inside amr_build_tree. Grows the local
         allocatable temporaries (t_parent, t_children, t_level, etc.)
         using Fortran move_alloc. These temporaries are discarded after
         the tree topology is copied to shared memory.


2. read_ramses_amr.f90
   Module: read_ramses_amr_mod

   Reads leaf-cell data from RAMSES AMR simulations and provides a generic
   text-format reader for other octree codes.

   ramses_read_leaf_cells(repository, snapnum, xleaf,yleaf,zleaf,
                          leaf_level, nH_arr, T_arr, vx/vy/vz_arr,
                          nleaf, boxlen_cm)
       Top-level RAMSES reader. Calls ramses_read_info to get unit scales,
       then ramses_count_leaves (pass 1) and ramses_read_all_cpus (pass 2).
       Outputs positions in physical units (cm), nH (cm^-3), T (K),
       velocities (cm/s).

   ramses_read_info(repository, snapnum, ncpu, unit_l, unit_d, unit_t,
                    unit_v, boxlen, gamma_eos)
       Parses info_{snapnum:05d}.txt for the number of CPUs, length unit
       (unit_l, cm), density unit (unit_d, g/cm^3), time unit (unit_t, s),
       velocity unit (unit_v, cm/s), box length (code units), and adiabatic
       index gamma.

   ramses_count_leaves(repository, snapnum, ncpu, unit_l, boxlen_code,
                       nleaf_total)
       First pass over all amr_*.out* files. Counts cells where son == 0
       (leaf cells) to determine the total number of leaves before
       allocating output arrays.

   ramses_read_all_cpus(repository, snapnum, ncpu, unit_l, unit_d, unit_v,
                        gamma_eos, boxlen_code, xleaf, yleaf, zleaf,
                        leaf_level, nH_arr, T_arr, vx/vy/vz_arr,
                        nleaf_total)
       Second pass. For each CPU file reads:
         - AMR header: ncpu, ndim, nx/ny/nz, nlevelmax, nboundary
         - ngridlevel, ngridfile, ngridbound arrays
         - Grid positions (xg) and son arrays per level
         - Hydro variables: density, momentum x/y/z, total energy
       Leaf identification: son == 0 for all 8 children of a cell.
       Position conversion:
         xleaf = (xg + xc - xbound) / nx + 0.5   (fraction of boxlen)
       Temperature from total energy:
         eint = (E_tot - 0.5*rho*v^2) / rho
         T = (gamma-1) * eint * unit_v^2 * mu*mH / kB
       where mu = 1.22 (mean molecular weight for ionised gas).

   generic_amr_read(filename, xleaf,yleaf,zleaf, leaf_level,
                    nH_arr, T_arr, vx/vy/vz_arr, nleaf, boxlen_phys)
       Reads a plain-text AMR file. Format:
         Line 1: nleaf  boxlen
         Lines 2..(nleaf+1):
           x  y  z  level  nH(cm^-3)  T(K)  vx(km/s)  vy(km/s)  vz(km/s)
       The position unit of x, y, z and boxlen is determined by
       par%distance_unit (kpc, pc, au, or dimensionless) exactly as in
       Cartesian mode. Velocities are in km/s (converted internally).

   skip_records(unit, n)
       Helper that reads and discards n Fortran unformatted records from
       the given unit number. Used to skip header records in RAMSES files.


3. raytrace_amr.f90
   Module: raytrace_amr_mod

   AMR ray-tracing routines with identical call signatures to the Cartesian
   procedure pointer targets in raytrace_car.f90. When
   par%use_amr_grid = .true., the procedure pointers are reassigned to these.

   raytrace_to_tau_amr(photon, grid, tau_in)
       Advances the photon until it has accumulated optical depth tau_in,
       or exits the box. At each step:
         1. amr_cell_exit gives t_exit and iface for the current leaf.
         2. tau_cell = rhokap(il) * voigt(xfreq, voigt_a(il)) * ds
                     + rhokapD(il) * ds    [if dust present]
         3. If tau_cell >= tau_remaining, interpolate exact crossing point
            and stop; otherwise advance to the face.
         4. Use amr_next_leaf (O(1) neighbor table lookup) to enter the
            next cell; no tree re-descent at each boundary crossing.
         5. At cell boundaries, apply Doppler frequency shift:
            xfreq_new = (xfreq_old + u_old) * Dfreq_old/Dfreq_new - u_new
       Escaped photons (iface = 0, outside box) are recorded in
       amr_grid%Jout with atomic update.
       Early exit when accumulated tau exceeds tau_huge = 745.2.

   raytrace_to_edge_amr(photon0, grid, tau)
       Returns total optical depth (gas + dust) from photon0's current
       position to the box boundary, without modifying photon0.

   raytrace_to_edge_tau_gas_amr
       Returns gas-only optical depth to the box boundary.

   raytrace_to_edge_column_amr
       Returns column density N_gas (cm^-2) and dust optical depth
       tau_dust separately, traversing to the box boundary.
       N_gas = sum( rhokap(il) * Dfreq(il) / sigma0 * ds )

   raytrace_to_dist_amr(photon0, grid, dist, tau)
       Returns total optical depth over a given distance dist (cm).

   raytrace_to_dist_tau_gas_amr
       Gas-only optical depth over a given distance.

   raytrace_to_dist_column_amr
       N_gas and tau_dust over a given distance.


4. scattering_amr.f90
   Module: scattering_amr_mod

   AMR scattering routines (Stokes and no-Stokes) that read physical
   quantities from amr_grid instead of the Cartesian grid arrays.
   Mirrors scattering_car.f90 in structure.

   scattering_amr_dispatch(photon, grid)
       Top-level dispatcher. Determines dust vs. resonance scattering
       probability using amr_grid%rhokapD(il) and amr_grid%rhokap(il),
       then calls scatter_dust or scatter_resonance via the procedure
       pointers (which have already been reassigned to the AMR variants).

   scatter_dust_nostokes_amr(photon, grid)  [no Stokes]
       Dust scattering event:
         - Stochastic absorption test (or reduced-weight albedo weighting).
         - Records absorbed weight in amr_grid%Jabs(ix) if save_Jabs.
         - Frequency shift to the fluid rest frame using amr_grid%vfx/vfy/vfz
           and amr_grid%Dfreq(il).
         - New direction sampled from Henyey-Greenstein phase function
           (isotropic if hgg = 0).

   scatter_dust_stokes_amr(photon, grid)  [Stokes, added 2026-04-23]
       Dust scattering with full Stokes vector update. See Update note above.

   scatter_resonance_nostokes_amr(photon, grid)  [no Stokes]
       Resonance (line) scattering event:
         - Calls do_resonance procedure pointer (reassigned to an AMR variant)
           to sample the scattering atom velocity uz and new direction
           cosines cost, sint.
         - phi (azimuthal angle) is drawn uniformly in [0, 2*pi); theta
           (polar angle between incoming and outgoing directions) is sampled
           from P(cos theta) = (3/8)*E1*cos^2(theta) + (4-E1)/8 via
           rand_resonance(E1), where E1 depends on the transition type.
         - Core-skip acceleration using amr_grid%xcrit / xcrit2.
         - Recoil correction: xfreq -= g_recoil*(1-cost) if par%recoil.
         - Updates photon direction vector.

   scatter_resonance_stokes_amr(photon, grid)  [Stokes, added 2026-04-23]
       Resonance scattering with full Stokes vector update. See Update note above.

   do_resonance1_amr(photon, grid, uz, xfreq_atom, cost, sint [, Sij])
       Single-line resonance (Ly-alpha H I, Fe II, etc.).
       Samples uz from rand_resonance_vz(xfreq, amr_grid%voigt_a(il)),
       sets xfreq_atom = xfreq - uz, samples cost from rand_resonance(E1).
       Optional Stokes matrix elements S11, S22, S12, S33, S44 returned
       if called with those arguments.

   do_resonance2_amr
       Two-line resonance (C II, Ca II H&K doublet, etc.).
       Selects the H or K component based on their relative Voigt profiles
       at the current frequency, then samples uz from the chosen component.
       Uses amr_grid%Dfreq(il) to convert DnuHK_Hz to Doppler units.

   do_resonance4_amr
       One upward transition, multiple downward transitions (e.g. Fe II 2374).
       Samples uz from rand_resonance_vz, then selects the downward transition
       branch via rand_alias_choise(line%b(1)%P_down, line%b(1)%A_down).
       Applies the frequency offset (Elow_Hz / Dfreq) for non-primary branches
       and sets E1/E2/E3 polarisation factors for the chosen branch.

   do_resonance5_amr
       Two upward transitions, multiple downward transitions (e.g. Fe II multi-level).
       Selects the upward branch (iup=1 or 2) by comparing Voigt-weighted oscillator
       strengths at the current frequency and at the second line offset (delE_Hz(2)).
       Then selects the downward branch via rand_alias_choise if ndown > 1.
       Applies Elow_Hz frequency offset and sets E1/E2/E3 for the chosen (iup,idown).

   do_resonance6_amr
       Three upward transitions, one downward each (e.g. three-level system).
       Selects the upward branch (iup=1,2,3) by cumulative Voigt-weighted
       oscillator strengths at offsets delE_Hz(2) and delE_Hz(3).
       Sets E1/E2/E3 from line%b(iup)%E1/E2/E3(1) for the chosen branch.


5. grid_mod_amr.f90
   Module: grid_mod_amr

   AMR grid creation and destruction. Called from main.f90 when
   par%use_amr_grid = .true.

   grid_create_amr(grid)
       Full AMR grid setup in nine steps:
         (1) Rank 0 reads leaf positions, levels, and physical data
             from disk (RAMSES or generic format, selected by par%amr_type).
         (2) Broadcasts all raw arrays to every MPI rank via MPI_BCAST.
         (3) Calls amr_build_tree to construct the octree from the leaf list,
             then amr_build_neighbors to precompute the face-neighbor table.
         (4) Computes reference Doppler frequency Dfreq_ref (at mean T),
             mean Voigt parameter voigt_amean, and mean Doppler width
             Dfreq_mean.
         (5) Allocates physical arrays via amr_alloc_phys.
         (6) For each leaf il:
               Dfreq(il)    = line%nu0 / c * sqrt(2kT/mH)
               voigt_a(il)  = line%A21 / (4*pi*Dfreq(il))
               Neutral fraction via CIE (Collisional Ionisation Equilibrium)
               rhokap(il)   = nH * f_neutral * sigma0 * Dfreq_ref/Dfreq(il)
               rhokapD(il)  = nH * mu_H * DGR * kappa_dust  (if DGR > 0)
               vfx/vfy/vfz  = bulk velocity / v_thermal
         (7) Normalises opacities to par%taumax (optical depth from box
             center to +z boundary along the z-axis, matching the Cartesian
             convention where taumax = tau from center to sphere surface).
         (8) Sets core-skip threshold:
               xcrit  = (amr_grid%voigt_amean / pi)^(1/3)
               xcrit2 = xcrit^2
         (9) Calls amr_setup_emissivity when par%emiss_file is set.
             Supported AMR diffuse-emissivity inputs:
               - txt/dat  : 1D plane/spherical emissivity profiles
               - density1 : emissivity proportional to rhokap
               - density2 : emissivity proportional to rhokap^2
             For leaf-based 3D sampling, the probability is weighted by the
             leaf cell volume so that photons are sampled from emissivity per
             unit volume rather than per cell. Composite-bias weights are
             built consistently with the nonuniform AMR leaf volumes.
         (10) Calls amr_setup_freq_grid to set up the output frequency axis.
         (11) Allocates output arrays via amr_alloc_output.

   grid_destroy_amr()
       Deallocates all AMR arrays (tree structure, physical fields, output).

   amr_setup_freq_grid(atau0, boxlen_cm)
       Determines xfreq_min and xfreq_max (either from par% values or
       auto-scaled from atau0 = (voigt_a * taumax)^(1/3)), sets dxfreq,
       and allocates xfreq, velocity, and wavelength arrays.

   output_normalize_amr(nphotons)
       Divides Jout by nphotons * dxfreq * 2*pi * area, where
       area = 4*pi * (L_box/2)^2 * distance2cm^2.
       This matches the Cartesian convention (4*pi * rmax^2 * distance2cm^2)
       when rmax = L_box/2.


6. peelingoff_amr.f90

   AMR peel-off observer calculation. Called by setup.f90 when
   par%use_amr_grid = .true. and par%save_peeloff = .true.

   peeling_dust_nostokes_amr(photon, grid)  [no Stokes]
       Peel-off contribution for dust scattering: computes the Henyey-
       Greenstein phase function weight for the observer direction, then
       calls raytrace_to_edge_amr to compute the optical depth to the box
       boundary, and records the attenuated contribution in the observer map.

   peeling_dust_stokes_amr(photon, grid)  [Stokes, added 2026-04-23]
       Peel-off with full Stokes I/Q/U/V output. See Update note above.

   peeling_resonance_nostokes_amr(photon, grid)  [no Stokes]
       Peel-off contribution for resonance scattering: computes the
       resonance phase function weight for the observer direction, applies
       the Doppler frequency shift for the observer geometry, calls
       raytrace_to_edge_amr, and records in the observer map.

   peeling_resonance_stokes_amr(photon, grid, xfreq_atom, vel_atom)  [Stokes, added 2026-04-23]
       Peel-off with full Stokes I/Q/U/V output. See Update note above.

   In subroutine scattering():
     Added at module level within the subroutine:
       use octree_mod, only: amr_grid

     Added an early AMR dispatch branch before the Cartesian code:
       if (par%use_amr_grid) then
          i1 = photon%icell_amr
          if (par%DGR > 0.0_wp .and. associated(amr_grid%rhokapD)) then
             p_dust = amr_grid%rhokapD(i1) / &
                 (amr_grid%rhokap(i1)*voigt(photon%xfreq,amr_grid%voigt_a(i1)) &
                  + amr_grid%rhokapD(i1))
             if (rand_number() <= p_dust) then

             endif
          else
             call scatter_resonance(photon,grid)
          endif

       endif

Bugs Fixed (Coordinate System / Physics)                     [2026-04-25]

4. read_ramses_amr.f90 -- generic FITS reader incorrectly shifted leaf coordinates
   Problem:  generic_amr_read_fits applied
               xleaf(:) = xleaf(:) - origin_x   (and y, z)
             where ORIGINX = -boxlen/2 (written by Python AMRGrid).
             This shifted coordinates from [-boxlen/2, +boxlen/2] to [0, boxlen],
             moving the grid centre from (0,0,0) to (boxlen/2, boxlen/2, boxlen/2).
             LaRT assumes (0,0,0) as the grid centre for peel-off geometry and
             the default point-source position (xs_point = ys_point = zs_point = 0).
             A point source at (0,0,0) was therefore placed at the corner of the
             sphere rather than at its centre, producing ~3x fewer scatterings
             and incorrect spectra.
   Fix:      Removed the three coordinate-shift lines in generic_amr_read_fits.
             Added a comment explaining that ORIGINX/Y/Z headers are informational
             only and must not be used to shift the already-centred coordinates.

5. read_ramses_amr.f90 -- RAMSES reader produced coordinates in [0, boxlen_cm]
   Problem:  ramses_read_leaf_cells converted fractional RAMSES positions
             (range [0,1]) to physical cm via   xl = xl * boxlen_cm,
             producing positions in [0, boxlen_cm]. This is inconsistent with
             the LaRT convention that the box is centred at the origin, and
             placed the RAMSES box corner at (0,0,0) instead of (-boxlen/2,...).
   Fix:      Changed to   xl = (xl - 0.5_wp) * boxlen_cm   so that RAMSES
             positions are mapped to [-boxlen_cm/2, +boxlen_cm/2].

6. grid_mod_amr.f90 -- amr_build_tree called with wrong box bounds
   Problem:  amr_build_tree was called with bounds [0, boxlen_code] on all
             three axes, so the octree root centre was at
             (boxlen/2, boxlen/2, boxlen/2) instead of (0,0,0). Even with
             correctly centred leaf coordinates the tree-descent would
             misidentify which child octant a photon belongs to for positions
             near the origin.
   Fix:      Changed the bounds to [-boxlen_code/2, +boxlen_code/2] on all
             three axes, matching the centred coordinate convention used by
             both the generic FITS reader and the RAMSES reader.

Input File Usage
--------------------------------------------------------------------------------

To enable AMR mode add the following to the input namelist file (.in):

  &parameters
   par%use_amr_grid = .true.

   ! RAMSES format
   par%amr_type    = 'ramses'
   par%amr_file    = '/path/to/simulation/output_00001'
   par%amr_snapnum = 1

   ! or generic text format
   ! par%amr_type      = 'generic'
   ! par%amr_file      = '/path/to/amr_data.dat'
   ! par%distance_unit = 'kpc'   ! unit of x,y,z and boxlen in the data file

   par%taumax   = 1.0e4     ! optical depth from box center to +z boundary
   par%nxfreq   = 201       ! number of output frequency channels
   ...
  /

Generic text file format (amr_type = 'generic'):
  Line 1:             nleaf  boxlen
  Lines 2..(nleaf+1): x  y  z  level  nH(cm^-3)  T(K)  vx(km/s)  vy(km/s)  vz(km/s)
  The unit of x, y, z and boxlen is set by par%distance_unit (kpc, pc, au,
  or dimensionless). Velocities are always in km/s.

Coordinate convention:
  All leaf positions must be in [-boxlen/2, +boxlen/2] on each axis.
  The box is centred at the origin (0,0,0).
  Python AMRGrid writes coordinates in this convention automatically.
  RAMSES fractional positions x in [0,1] are converted to (x - 0.5) * boxlen.

Tau normalization convention:
  taumax = optical depth from box center to the +z boundary along the z-axis.
  This matches the Cartesian convention where taumax = tau from center to
  sphere surface. Comparison Cartesian runs should use rmax = L_box/2.

Jout normalization:
  Jout is divided by nphotons * dxfreq * 2*pi * 4*pi*(L_box/2)^2 * distance2cm^2.
  This matches the Cartesian convention when rmax = L_box/2.

--------------------------------------------------------------------------------
Design Notes
--------------------------------------------------------------------------------

Procedure pointer dispatch
  LaRT uses Fortran procedure pointers (raytrace_to_tau, scatter_resonance,
  do_resonance, etc.) declared in define.f90. setup.f90 assigns the correct
  targets at startup. When par%use_amr_grid = .true., all pointers are
  redirected to the AMR variants and the routine returns immediately, so none
  of the Cartesian targets are ever called in AMR mode.

Octree storage
  The octree is stored as a set of flat integer/real arrays indexed by cell
  number (1-based). The root is cell 1. Child ordering follows the standard
  octant convention: child_index = 1 + ix + 2*iy + 4*iz, where ix=1 if
  x >= cell centre x, and similarly for iy, iz. Leaf cells have all eight
  children equal to zero.

O(1) cell boundary crossings
  After amr_build_tree, amr_build_neighbors precomputes a face-neighbor table
  (6 entries per leaf, one per face). amr_next_leaf then performs O(1) lookups
  at each cell boundary without re-descending the tree.

Frequency shift at cell boundaries
  When a photon crosses from leaf il_old to leaf il_new, its dimensionless
  frequency is updated as:
    xfreq_new = (xfreq_old + u_dot_k_old) * Dfreq_old/Dfreq_new - u_dot_k_new
  where u_dot_k = vfx*kx + vfy*ky + vfz*kz is the line-of-sight bulk
  velocity in thermal units. This preserves the photon's physical frequency.

MPI distribution
  Rank 0 reads all leaf data from disk, then broadcasts the raw arrays to
  all other ranks via MPI_BCAST. Every rank builds the same octree
  independently from the broadcast data. The completed tree topology and
  all physical arrays are then stored in MPI-3 shared-memory windows
  (create_shared_mem / destroy_mem in memory_mod_mpi.f90), giving one
  physical copy per shared-memory node. All ranks on a node map the same
  pages, so per-node memory scales with data size rather than with the
  number of ranks per node. Only the spectral output arrays (Jout, Jin,
  Jabs) are per-rank regular allocatables, accumulated independently and
  reduced at the end.

Thread safety
  The AMR leaf arrays are read-only during the photon loop. Only the Jout
  and Jabs output accumulation arrays are written, using !$OMP ATOMIC UPDATE
  to ensure correctness in OpenMP mode.

Extensibility
  To support a new AMR data format, write a reader subroutine in (or beside)
  read_ramses_amr.f90 that fills the same output arrays (xleaf, yleaf, zleaf,
  leaf_level, nH_arr, T_arr, vx/vy/vz_arr, nleaf, boxlen_phys), then add a
  new case branch in grid_create_amr (grid_mod_amr.f90) keyed on par%amr_type.
  No other files need to change.

Validation
--------------------------------------------------------------------------------

AMR sphere tests vs. equivalent Cartesian runs (T=10^4 K, point source at
box center, dimensionless units with rmax = L_box/2 = 50):

  tau=100:  Jout_max AMR = 6.630e-03, Cartesian = 6.836e-03, ratio = 0.970
  tau=1e4:  Jout_max AMR = 7.144e-03, Cartesian = 7.402e-03, ratio = 0.965
  Peak velocity: exact match at both optical depths.

The ~3-5% amplitude difference reflects the octree volume-averaging of the
sphere boundary (voxelization artifact), not a normalization error.
