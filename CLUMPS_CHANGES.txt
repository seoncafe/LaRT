REFACTOR: cl_vx/y/z stored as v / cl_vtherm (dimensionless), matching the
          Cartesian/AMR grid%vfx convention.

  Previously cl_vx/cl_vy/cl_vz were stored in km/s, requiring a per-call
  division by cl_vtherm in raytrace_clump.f90 (6 places) and peelingoff_rect.f90
  (4 places).  This was inconsistent with the Cartesian/AMR convention, where
  grid%vfx is already pre-normalised to v/vtherm in grid_create_car
  (cf. grid_mod_car.f90, where each cell's v_field is divided by the local
  vtherm right after being read).

  Changes:
    - clump_mod.f90: after the MPI_BCAST of cl_vx/vy/vz in init_clumps(),
      divide each component by cl_vtherm (h_rank=0 only, matching the shared-
      memory ownership).  Updated the declaration comment for cl_vx/y/z.
    - clump_mod.f90 / write_clumps_fits: multiply by cl_vtherm when writing
      VX/VY/VZ columns so the user-facing FITS file remains in km/s.
    - raytrace_clump.f90: removed `/ cl_vtherm` from all 6 occurrences of
      `u_los = (cl_vx*kx + cl_vy*ky + cl_vz*kz) / cl_vtherm` patterns.
      Updated the frequency-convention comment block.
    - peelingoff_rect.f90 (v2.00 + v2.10): removed `/ cl_vtherm` from all
      4 occurrences in the clump-aware u1 expressions; removed cl_vtherm
      from the `use clump_mod, only:` clause.

  Effect: small per-call savings in tight loops; no change in physics.
  cl_vtherm is a scalar so the previous division was already nearly free
  for the compiler, but the new convention is cleaner and removes a class
  of "forgot to divide by cl_vtherm" bugs.  Verified with fcov=1 and fcov=5
  rotating-galaxy-halo cases (Direct/Jin within statistical agreement of
  the Poisson exp(-fcov) expectation).

  Files modified: clump_mod.f90, raytrace_clump.f90, peelingoff_rect.f90
                  (v2.00 and v2.10)


BUG FIX (A): FITS WCS axis labels swapped in the 3D peel-off cube.

  obs%scatt is allocated as obs%scatt(par%nxfreq, par%nxim, par%nyim), so
  in the FITS file:
    NAXIS1 = nxfreq  (frequency / wavelength)
    NAXIS2 = nxim    (RA pixel)
    NAXIS3 = nyim    (DEC pixel)

  The previous code wrote
      CTYPE1='RA--TAN', CD1_1=par%dxim, CRPIX1=(nxim+1)/2
      CTYPE2='DEC-TAN', CD2_2=par%dyim, CRPIX2=(nyim+1)/2
  i.e. the spatial WCS was attached to the (wavelength) NAXIS1 axis, and
  NAXIS3 had no WCS at all.  External tools (astropy.wcs, ds9) consequently
  saw a 400-pixel "RA" axis and an unidentified third axis.

  Fix (write_output_rect.f90, both `Scattered` and `Stokes_I` 3D HDUs in
  write_output_peeling_3D):
    NAXIS1 -> CTYPE1='WAVE', CUNIT1='Angstrom',
              CRPIX1=1, CRVAL1=grid%wavelength(1), CD1_1=-grid%dwave
              (CD1_1 is negative because xfreq increases with i and
               wavelength = (1 - vtherm*xfreq/c)*lambda0 decreases with i)
    NAXIS2 -> CTYPE2='RA--TAN',  CUNIT2='deg', CD2_2=par%dxim, CRPIX2=(nxim+1)/2
    NAXIS3 -> CTYPE3='DEC-TAN', CUNIT3='deg', CD3_3=par%dyim, CRPIX3=(nyim+1)/2
  The previously-written DXFREQ, DWAVE, XFREQ1, XFREQ2 scalar keywords
  are kept for backward compatibility with existing analysis scripts.

  Verified: astropy.wcs.WCS(header).wcs.ctype now returns
            ['WAVE', 'RA--TAN', 'DEC-TAN']; pixel ~200 maps to 1215.67 A
            (Lyalpha vacuum wavelength) for the fcov1 test.

  Files modified: write_output_rect.f90 (v2.00 and v2.10)


BUG FIX (B): par%tauhomo / par%taumax / par%N_gashomo / par%N_gasmax were
             derived from a unit-density placeholder grid in clump mode.

  In grid_create_clump(), the call sequence used to be
      call grid_create(grid)      ! computes tauhomo, taumax, N_gashomo,
                                  ! N_gasmax from a uniform grid%rhokap=1
      call init_clumps(par%rmax)  ! computes the actual clump cl_rhokap
      grid%rhokap = 0             ! override (clump mode has zero ICM opacity)
  so the derived "homogeneous" quantities had no physical meaning (e.g.
  TAUHOMO ~ 5.9e-14 in the FITS header), and the auto-detection of the
  frequency range inside car_setup_freq_grid (via xscale * atau3) got a
  meaningless atau3.

  Fix (grid_mod_clump.f90): swap the order so init_clumps() runs first,
  then derive the proper clump-medium values BEFORE grid_create() is
  called.  Only set them when the user has not already specified the
  parameter (i.e. par%* <= 0):

    voigt0           = voigt(0, cl_voigt_a)
    f_vol            = N_clumps * (r_c/R_max)^3
    f_cov            = (3/4) * N_clumps * (r_c/R_max)^2     (LaRT convention:
                                  expected number of clump crossings per
                                  radial path from centre to surface)
    tau_per_clump_lc = cl_rhokap * voigt0 * r_c            (line-centre
                                  optical depth of one clump, centre to
                                  surface)
    par%tauhomo      = (4/3) * f_cov * tau_per_clump_lc    (= homogeneous-
                                  equivalent line-centre tau, identical
                                  to the EXPECTED radial taumax for a
                                  random sightline through the clumpy sphere)
    par%taumax       = (4/3) * f_cov * tau_per_clump_lc    (no physical
                                  meaning for a single LOS in a clumpy
                                  medium; set equal to tauhomo so the
                                  xscale auto-detection picks a sensible
                                  bracket)
    par%N_gashomo    = (4/3) * f_cov * (cl_rhokap/line%cross0) * cl_Dfreq * r_c
    par%N_gasmax     = (4/3) * f_cov * (cl_rhokap/line%cross0) * cl_Dfreq * r_c

  The factor (4/3) f_cov reduces to the trivial result
      N_HI_homo = (4/3) * f_cov * clump_NHI
  when par%clump_NHI is the input.

  Verified for fcov1, NHI=7.5e17, T=1e4:
      TAUHOMO = 5.90e4   (= 1.0e18 * 5.88e-14 cm^2 = N_HI_homo * sigma_0)
      ATAU3   = 3.03     (consistent with auto-detected xfreq_max ~ 16,
                           covering the +/- 200 km/s rotation range)

  Files modified: grid_mod_clump.f90 (v2.00 and v2.10)


BUG FIX (C): Peel-off frame inconsistency for clump mode with non-zero
             clump bulk velocities.

  When a photon scatters inside a clump, photon%xfreq is in the CLUMP frame.
  The Jout binning (raytrace_clump.f90) shifts to the lab frame at clump
  exit (xfreq += u_los), so Jout is always in the lab frame.  The peel-off
  binning (peeling_*_outside in peelingoff_rect.f90), however, used
      u1 = grid%vfx(...) * pobs%kx + grid%vfy(...) * pobs%ky + grid%vfz(...)*pobs%kz
  to convert to the lab frame, and grid%vfx/vfy/vfz are forced to 0 in
  clump mode.  Hence u1 = 0 and the peel-off photon was binned at the
  CLUMP-frame frequency, while Jout was binned at the LAB-frame frequency.

  Symptom: for runs with significant clump bulk velocities (e.g.
           rotating_galaxy_halo with Vrot=300 km/s), the peel-off spectrum
           was systematically narrower than Jout — the rotational broadening
           of the line was missing from the peel-off output.

  Fix: in each of peeling_dust_stokes_outside, peeling_dust_nostokes_outside,
       peeling_resonance_stokes_outside, peeling_resonance_nostokes_outside,
       branch on `par%use_clump_medium .and. photon%icell_clump > 0` and
       use the host clump's velocity instead of grid%vf:

           u1 = cl_vx(photon%icell_clump) * pobs%kx
              + cl_vy(photon%icell_clump) * pobs%ky
              + cl_vz(photon%icell_clump) * pobs%kz

       (cl_v* are in v/vtherm units after the refactor above; with the
       previous km/s storage the formula was the same expression divided
       by cl_vtherm.)  The `peeling_direct_outside` routine is left alone
       because at photon birth photon%icell_clump = 0 (the photon is in
       vacuum) and the existing u1 = grid%vfx*... = 0 already gives the
       correct lab-frame xfreq.

  Verified: with the fix, the peel-off scattered spectra of fcov1 broaden
            to span +/- ~200 km/s with a peak that shifts with the
            line-of-sight component of the host clumps' rotation velocity,
            matching the angular structure expected from the rotating-halo
            velocity field.

  Files modified: peelingoff_rect.f90, setup.f90 (clump-mode comment update)
                  (v2.00 and v2.10)

BUG FIX: cl_rhokap wrong for clump_NHI and clump_nH paths (missing / cl_Dfreq)

  In init_clumps(), the opacity derivation for the clump_NHI and clump_nH
  input paths produced cl_rhokap with units of [Hz/code_unit] instead of the
  required [code_unit^-1]:

    line%cross0 = sigma_0 * f_osc / sqrt(pi)   [cm^2*Hz]   (NOT [cm^2])

  The actual line-center cross-section [cm^2] is  line%cross0 / cl_Dfreq.
  Both paths were missing a division by cl_Dfreq:

  WRONG:
    clump_NHI:  cl_rhokap = par%clump_NHI * line%cross0 / cl_radius
    clump_nH:   cl_rhokap = par%clump_nH  * line%cross0 * par%distance2cm

  CORRECT:
    clump_NHI:  cl_rhokap = par%clump_NHI * line%cross0 / (cl_Dfreq * cl_radius)
    clump_nH:   cl_rhokap = par%clump_nH  * line%cross0 * par%distance2cm / cl_Dfreq

  Effect of the bug: cl_rhokap was ~cl_Dfreq (~10^11) times too large, making
  the per-clump tau0 effectively infinite (e.g. ~4.7e15 instead of ~4.4e4 for
  clump_NHI=7.5e17 cm^-2, r_c=0.001, T=1e4 K).  Photons could never escape
  individual clumps, causing the simulation to stall indefinitely.

  Note: the clump_tau0 path was unaffected (it derives cl_rhokap from a
  dimensionless tau directly and uses no cross0).

  Files modified: clump_mod.f90  (v2.00 and v2.10)

NEW FEATURE: clump_NHI — per-clump HI column density input
  - Added par%clump_NHI (real, default -1; units: cm^-2).

    Specifies the HI column density of an INDIVIDUAL CLUMP from its centre to
    its surface.  This quantity must be carefully distinguished from the total
    SYSTEM column density (par%N_HImax), which is the average HI column density
    along a radial sightline through the entire clumpy sphere (centre to outer
    boundary, averaged over the random clump population).

    The per-clump opacity is derived as:
      cl_rhokap = clump_NHI * (line%cross0/cl_Dfreq) / cl_radius   [code_unit^-1]
    which is equivalent to:
      tau0 = clump_NHI * sigma0 * phi(0, a_V^cl)
    where sigma0 = line%cross0/cl_Dfreq is the line-centre cross-section [cm^2]
    and phi is the Voigt profile value at x=0.
    (Note: line%cross0 has units [cm^2*Hz]; the physical cross-section [cm^2]
     is line%cross0 / cl_Dfreq.)

    Unlike clump_nH (which requires par%distance_unit to convert number density
    [cm^-3] to opacity per code unit), clump_NHI [cm^-2] is a physical column
    density and does NOT require distance_unit.

    Priority order when deriving cl_rhokap (first positive value wins):
      clump_tau0  > clump_NHI  > clump_nH

  - FITS keyword IN_NHI added to write_clumps_fits (BinTable HDU).

  Files modified: define.f90, clump_mod.f90  (v2.00 and v2.10)

OVERVIEW
--------
Added a clumpy medium radiative transfer mode (par%use_clump_medium = .true.).
The medium consists of N non-overlapping spherical clumps of radius r_c placed
uniformly at random inside a sphere of radius R.  Inter-clump space is vacuum.
Clumps have uniform opacity, temperature, and bulk velocities that are the sum
of (1) a position-dependent systematic flow (par%velocity_type) and (2) an
independent Gaussian random component (par%clump_sigma_v).  Either may be zero.

NEW FILES
---------
  clump_mod.f90
    - Module clump_mod: all clump physics and data.
    - init_clumps(R_sphere): derives N_clumps from input (f_cov / f_vol /
      N_clumps direct), computes cl_Dfreq / cl_voigt_a / cl_rhokap, allocates
      MPI-3 shared memory, runs RSA placement, builds CSR grid.
    - generate_clumps(): RSA with linked-list hash grid (O(N) expected).
      Assigns Gaussian random velocity per component if par%clump_sigma_v > 0.
    - assign_clump_velocities_from_type(): adds systematic flow velocity
      (from par%velocity_type) to each clump's existing velocity.
      Called after generate_clumps() when par%velocity_type is non-empty.
      Supported types: 'hubble', 'constant_radial', 'parallel_velocity',
      'ssh', 'rotating_solid_body', 'rotating_galaxy_halo'.
      Uses same parameter names (Vexp, Vpeak, rpeak, DeltaV, Vrot, rinner,
      Vx/Vy/Vz) as the Cartesian grid velocity assignment in grid_mod_car.f90.
    - build_clump_csr(): two-pass CSR construction in MPI shared memory.
    - find_next_clump(): DDA (Amanatides & Woo 1987) through CSR grid;
      returns nearest clump hit along the ray.
    - ray_sphere_isect(): ray–sphere intersection test.
    - clump_exit_dist(): distance to clump exit from interior point.
    - sphere_exit_dist(): distance to outer sphere exit.
    - destroy_clumps(): collective MPI_WIN_FREE + nullify.
    - write_clumps_fits(fname): writes clump positions/velocities/parameters
      to a FITS binary table (called when par%save_clump_info = .true.).
    - Module-level scalars: N_clumps, sphere_R, cl_radius, cl_rhokap,
      cl_voigt_a, cl_Dfreq, cl_vtherm, cl_temperature.
    - Shared-memory arrays: cl_x/y/z, cl_vx/vy/vz (positions/velocities),
      cg_start/cg_list (CSR acceleration grid).

  raytrace_clump.f90
    - Module raytrace_clump_mod: photon propagation through the clump medium.
    - raytrace_to_tau_clump: advances photon until tau_in is reached or photon
      exits outer sphere. Handles photon inside/outside clump states, frequency
      shifts at clump entry/exit, and Jout accumulation upon sphere exit.
    - raytrace_to_edge_clump: computes tau from current position to sphere
      boundary without moving the photon (read-only, used by peel-off and
      forced-first-scatter).
    - raytrace_to_edge_tau_gas_clump, raytrace_to_edge_column_clump,
      raytrace_to_dist_clump, raytrace_to_dist_tau_gas_clump,
      raytrace_to_dist_column_clump: additional raytrace variants for column
      density and sightline calculations.
    - update_cell_idx: internal helper to update photon%icell/jcell/kcell
      from photon position using floor() mapping.

  grid_mod_clump.f90
    - Module grid_mod_clump.
    - grid_create_clump(grid): forces bounding box to [-rmax, rmax]^3,
      calls grid_create(), then init_clumps(), sets grid%rhokap=0 /
      Dfreq=cl_Dfreq / voigt_a=cl_voigt_a / vf=0.
    - grid_destroy_clump(grid): calls grid_destroy() + destroy_clumps().

ALGORITHMS
----------
  Placement (RSA):
    Random Sequential Addition with linked-list hash grid.
    Grid size: M = min(512, floor(N^{1/3}) + 1) per dimension.
    Each candidate is drawn uniformly inside the sphere; accepted if no
    existing clump within 2*r_c.  27-neighbour cell search ensures O(1)
    expected cost per attempt for f_vol < 35%.

  Acceleration grid (CSR):
    Two-pass construction (count then fill).
    Cell size: (2*(R+r_c)) / M.
    Each clump registered in all cells whose AABB overlaps the clump sphere.
    Arrays in MPI-3 shared memory.

  Raytrace (DDA):
    Amanatides & Woo (1987) traversal of CSR grid.
    For each cell, test all registered clumps with ray–sphere intersection.
    Stop when cell entry distance > best hit distance (early termination).
    Frequency shifts applied at clump entry (subtract v_los/v_th) and exit
    (add v_los/v_th) to maintain correct rest-frame opacity.

BUGS FIXED
----------
  2026-04-24  generate_clumps called independently on each node (multi-node bug)
    - init_clumps guarded clump generation with h_rank==0.  h_rank is the
      rank within a node (hostcomm), so each node had its own h_rank=0 running
      an independent RSA pass with different random numbers.  On a multi-node
      run (e.g. lart1 + lart2 + lart3) this produced a different clump layout
      per node, corrupting photon transport.
    - Fix: only p_rank==0 (the single global rank 0) generates clumps.
      The resulting cl_x/y/z and cl_vx/vy/vz arrays are then broadcast from
      rank 0 of SAME_HRANK_COMM (= p_rank==0) to all other h_rank=0 processes
      (one per node) via six MPI_BCAST calls on mpar%SAME_HRANK_COMM.
      The subsequent CSR build is deterministic from positions, so every node
      constructs the same acceleration grid without any additional broadcast.
    - File: clump_mod.f90

  2026-04-24  Jout accumulation missing in raytrace_to_tau_clump
    - raytrace_to_tau_car accumulates grid%Jout(ix) += photon%wgt when the
      photon exits the grid boundary.  The clump version lacked this.
    - Result: Jout only received the direct (unscattered) contribution from
      add_escaped_fraction_to_Jout in run_simulation_mod; scattered-then-
      escaped photons were not counted.
    - Fix: added Jout += photon%wgt at all three exit paths in
      raytrace_to_tau_clump (clump-then-sphere exit, already-outside-sphere,
      no-clump-found-skip-to-sphere).
    - File: raytrace_clump.f90

  2026-04-24  FITS keyword names > 8 characters in write_clumps_fits
    - CFITSIO silently fails (non-zero status) on keyword names > 8 chars.
      Once status becomes non-zero, all subsequent CFITSIO calls are skipped,
      producing an unreadable FITS file.
    - Offending names: CL_RHOKAP (9 chars) -> RHOKAP (6),
                       CL_VTHERM (9 chars) -> VTHERM (6).
    - File: clump_mod.f90

  2026-04-24  Keywords written before BinTable HDU existed
    - fits_put_keyword was called on the primary HDU before
      fits_write_table_column had created the BinTable extension.
    - Fix: write all table columns first (creating the BinTable HDU), then
      write keywords into that same HDU.
    - File: clump_mod.f90

  2026-04-24  f_cov formula missing factor 3/4 in write_clumps_fits
    - The correct formula is f_cov = (3/4) * N * (r_c/R)^2 (radial path from
      center to surface).  The realised f_cov stored in the FITS header was
      missing the 3/4 factor.
    - Fix: f_cov_actual = 0.75 * real(ncl) * (cl_radius/sphere_R)**2.
    - File: clump_mod.f90

  2026-04-24  Unnecessary fourpi_over3 variable in write_clumps_fits
    - f_vol = N * (4pi/3 * r_c^3) / (4pi/3 * R^3) = N * (r_c/R)^3.
      Simplified to: f_vol_actual = real(ncl) * (cl_radius/sphere_R)**3.
    - File: clump_mod.f90

  2026-04-24  destroy_clumps used destroy_mem on non-root MPI ranks
    - destroy_mem uses MPI_WIN_SHARED_QUERY with rank=0 to find the base
      address, which differs from the local pointer on non-root ranks,
      causing a segfault on exit.
    - Fix: use destroy_shared_mem_all() (collective MPI_WIN_FREE) then
      nullify all pointers manually.
    - File: clump_mod.f90

  2026-04-24  use memory_mod_mpi instead of use memory_mod
    - The memory module is named memory_mod (not memory_mod_mpi).
    - File: clump_mod.f90
